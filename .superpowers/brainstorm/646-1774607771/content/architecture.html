System Architecture — Abstract to Concrete

Full pipeline from world files to browser

1

World files — source of truth

worlds/billings-montana/**/*.md — characters, scenes, timeline, clock. Written by world-clock and character-updater skills.

↓
2

LLM conversion skill — runs after every tick

Reads all .md files, outputs versioned JSON to dist/. Schema version embedded in every file. ⚠ Pitfall: schema drift

↓
3

Upload to bucket — JSON + images

JSON files + generated images pushed to object storage (GCS / S3 / R2). Each tick gets its own snapshot path. Old ticks stay — that's how time travel works. ⚠ Pitfall: bucket layout

↓
4

Cloud function (Go) — serves the app shell

Serves the compiled React + Tailwind bundle (embedded). No database — all data lives in the bucket. Function is stateless. ⚠ Pitfall: cold start

↓
5

React frontend — fetches JSON from bucket

Fetches JSON directly from bucket URLs. Timeline scrubber changes the tick path → everything re-renders. Map, characters, scenes all driven by the same JSON snapshot.

Pitfalls marked ⚠ — discuss in terminal
