Everything we need to build, abstract to concrete

Layer 1 — Data Pipeline
1
JSON Schema Spec
Define the output format once. schema_version field in every file. This is the contract between the LLM skill and the frontend.
2
LLM Conversion Skill
Runs after each world-clock tick. Reads worlds/**/*.md → writes versioned JSON to dist/ticks/{timestamp}/. Owns the schema.
3
Upload Skill
Pushes dist/ to GCS. Updates gs://bucket/index.json (master list of all ticks). Old ticks are never deleted — that's time travel.
GCS bucket layout
gs://world-bucket/
  index.json                  ← list of all ticks + metadata
  ticks/2026-03-26-0900/
    world.json               ← clock, locations, active events
    characters/jolene-voss.json
    characters/amber-tuck.json
    scenes/2026-03-26-0900.json
  ticks/2026-03-26-2000/     ← next tick, same structure
  assets/
    map.svg                   ← fictional world map
    portraits/jolene-voss.jpg
Layer 2 — The App
4
Custom SVG Map
Fictional illustrated map. Locations have named coordinates. Character pins move per tick. Stored in GCS, loaded once. Can evolve as the world grows.
5
Go 1.26 App (Cloud Run)
Single binary. Embeds the compiled React + Tailwind bundle. Serves the app shell. Stateless — all data comes from GCS. Cold start is fine.
6
React Frontend
Loads index.json → timeline scrubber. Select a tick → fetch that tick's JSON. Three panels: fictional map with pins, character sidebar, scene/diary reader.
Layer 3 — GCP Setup
7
GCS bucket + Cloud Run service + IAM
Public bucket (or CDN) for JSON + assets. Cloud Run for the app shell. Service account for the upload skill. One-time setup.
Recommended build order
① JSON schema spec  →  ② LLM conversion skill  →  ③ Upload skill + GCS setup  →  ④ Go app skeleton + React shell  →  ⑤ Timeline + character panel  →  ⑥ SVG map + pins  →  ⑦ Deploy to Cloud Run
