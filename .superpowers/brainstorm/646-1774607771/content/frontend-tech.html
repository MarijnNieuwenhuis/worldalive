What serves the frontend?

Go handles the .md → JSON conversion. What delivers the actual website?

A

Go serves everything

One Go binary: converts .md → JSON and serves the site. Run it, open browser. Single executable, zero dependencies. Go templates for HTML, vanilla JS for interactivity.

Pros

  • One thing to run
  • No npm, no build tools

Cons

  • Less interactive UI
B

Go converts, browser reads JSON directly

Go writes JSON files to a folder. A plain HTML/JS site reads those files directly — no server needed at runtime. Open index.html in a browser, done.

Pros

  • No server to run
  • Completely portable

Cons

  • Browser CORS limits on local files
C

Go converts + serves API, JS frontend fetches

Go binary serves JSON over HTTP. A richer JS frontend (vanilla or lightweight framework) fetches and renders it. Most flexible for a dynamic UI with time travel.

Pros

  • Cleanest time travel UI
  • Easy to extend

Cons

  • Still one binary, but two moving parts conceptually
