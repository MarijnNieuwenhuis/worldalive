How does the site stay in sync with the world?

When world-clock advances and new .md files are written, how does the site pick it up?

A

Live server — reads files on demand

Go app runs continuously, reads .md files when you request a page. Always fresh. No build step. Slightly slower per request but zero maintenance.

B

Build step — .md → JSON on clock advance

After each world-clock tick, run a converter that writes JSON files. Go app serves those static files. Fast, portable, easy to deploy anywhere.

C

Watcher — auto-rebuilds when files change

Go app watches the worlds/ directory. When any .md changes, it rebuilds the JSON automatically. Best of both — live feel, static serving speed.
